1. Portabilidade: Independência de Modelo

O "Segredo" da Longevidade: Pesos de modelos de IA são voláteis, mas o Dataset é eterno. Se decidirmos trocar o PaliGemma pelo Florence (ou por um modelo que ainda nem foi lançado), o "background de treinamento" não é perdido. O Universal Training Bucket (nosso dataset agnóstico) é usado para re-executar o script de fine-tuning no novo modelo, transferindo todo o conhecimento acumulado pelos revisores humanos para a nova arquitetura em poucas horas.

2. Estratégia Híbrida e Resiliência

Cloud API Fallback

Caso o volume de fichas supere a capacidade da GPU local ou o hardware sofra uma falha, o sistema roteia automaticamente a demanda para APIs externas. A inteligência do CISC não para.

Treino Híbrido

O Training Bucket captura dados tanto do fluxo local quanto do fluxo via API. Isso garante que, mesmo usando a nuvem temporariamente, o nosso modelo local continue aprendendo e se preparando para retomar a soberania total.

3. Requisitos de Hardware

📟 Recomendado (2026)

  • GPU: NVIDIA RTX 4070 Ti Super (16GB VRAM)
  • VRAM: Uso de bitsandbytes (NF4) para liberar espaço aos gradientes de treino.
  • RAM: 32GB DDR5
  • Armazenamento: NVMe Gen4 (Leitura rápida do Bucket).
  • NPU: Uso da NPU local para pré-processamento de imagem.

💡 Dica de Implementação

"Para o início da operação, se o dataset for escasso, utilizaremos o modelo para gerar Dados Sintéticos baseados nos templates das fichas da SESMA, acelerando o 'aquecimento' do fine-tuning antes do primeiro grande lote de HITL."
