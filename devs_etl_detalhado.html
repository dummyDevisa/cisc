Detalhamento Técnico do Processo

📄 1. Ingestão do Caos

O processo começa onde a saúde pública realmente acontece: no papel. Fichas de notificação e planilhas avulsas são coletadas. O desafio aqui é a variabilidade de formatos e a qualidade da caligrafia.

🤖 2. Visão Computacional

As imagens são processadas por redes neurais que analisam cada campo individualmente (nome, data, CID, endereço). Em vez de uma nota para o papel todo, a IA atribui um Índice de Confiança por célula, isolando incertezas de forma granular.

👀 3. Revisão Cirúrgica (HITL)

A interface de revisão não exige a releitura do documento completo. O sistema destaca apenas os campos duvidosos para o servidor do DEVS. O humano atua de forma cirúrgica, validando apenas as células onde a máquina sinalizou baixa confiança, o que reduz drasticamente o tempo de processamento.

⚙️ 4. Refinaria de Dados (Python)

Com os dados já digitais, entra em cena o Python. O pipeline executa deduplicação (evitar contar o mesmo caso duas vezes) e validação cruzada com as bases da Receita e do Cartão SUS.

📍 5. Normalização Geográfica

O dado é "limpo" territorialmente. Nomes de ruas e bairros grafados incorretamente são corrigidos para padrões oficiais do município, permitindo o cruzamento exato com mapas de risco climático.

🐘 6. Carga no DW/Datalake

O resultado final é injetado no coração do servidor. A partir daqui, o dado do DEVS está pronto para ser consumido pela Sala de Situação e cruzado com fontes agnósticas (CEMADEN, INMET).
